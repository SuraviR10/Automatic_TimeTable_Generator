===============================================================================
ALL 7 CONSTRAINTS ENFORCED
===============================================================================

1. STRICT WEEKLY HOURS
   Each subject placed exactly as specified in weekly_hours field
   Example: DBMS with 3 hours → Placed exactly 3 times in the week

2. DAILY DIVERSITY
   Same subject cannot appear twice on the same day
   Example: Monday has DBMS at P1 → DBMS cannot be at P3 on Monday

3. LAB CONTINUITY
   Labs always in 2 consecutive periods, never crossing breaks
   Valid slots: P1-P2, P3-P4, P5-P6
   Invalid: P2-P3 (crosses tea break), P4-P5 (crosses lunch break)

4. NSS/FREE → P6 ONLY
   NSS and FREE subjects only in last period
   Example: NSS can only be at P6, never at P1-P5

5. GLOBAL FACULTY CONFLICT
   Faculty cannot teach two classes simultaneously
   Tracked across ALL departments and sections
   Example: Dr. Smith teaching ISE-A at Monday P1 → Cannot teach CSE-B at Monday P1

6. LAB ROOM CLASH PREVENTION
   Same lab room cannot be used by multiple classes at same time
   Tracked globally across all sections
   Example: ISE-LAB-01 used by ISE-A at Monday P1-P2 → Cannot be used by ISE-B at Monday P1-P2

7. OPEN ELECTIVE SAME-SLOT
   OE subjects locked to SAME time slot across ALL departments
   Admin sets the locked day+slot
   Example: OE601 locked to Thursday P3 → All departments have OE601 at Thursday P3

TIMETABLE GENERATION FLOW
===============================================================================

STEP 0: Pre-fill Open Electives (OE Mega-Constraint)
- Load OE constraints from database
- Place OE subjects at locked day+slot
- Mark faculty as occupied
- Mark day for subject (daily diversity)

STEP 1: Place NSS/FREE in Period 6 Only
- Find available P6 slots
- Place NSS/FREE subjects
- Mark faculty as occupied

STEP 2: Place Labs (2-Hour Continuous Blocks)
- Get valid lab slots (P1-P2, P3-P4, P5-P6)
- For each lab:
  - Find available 2-hour slot
  - Check daily diversity (not already on this day)
  - Check faculty conflicts (both periods)
  - Get available lab room (no clash)
  - Place lab in both periods
  - Mark faculty and lab room as occupied

STEP 3: Place Theory Subjects
- For each theory subject:
  - Place weekly_hours times
  - Check daily diversity (not already on this day)
  - Check faculty conflicts
  - Place in available slot
  - Mark faculty as occupied

STEP 4: Validate All Constraints
- Check strict weekly hours
- Check daily diversity
- Check lab continuity
- Check NSS/FREE in P6
- Return violations if any

RETRY LOGIC:
- If generation fails, retry up to 5 times
- Clear occupancy between attempts
- Randomize placement for better success
- Return error if all attempts fail

TROUBLESHOOTING
===============================================================================

ISSUE: Buttons don't respond when clicked
SOLUTION:
1. Clear browser cache: Ctrl + Shift + Delete → All time → Clear data
2. Close ALL browser windows
3. Restart browser
4. Go to http://localhost:5000
5. Should work now

ISSUE: Server won't start
SOLUTION:
1. Check Python: py --version (should show 3.11+)
2. Check port: netstat -ano | findstr :5000
3. If port busy: Kill process or use different port
4. Reinstall dependencies: pip install -r requirements.txt

ISSUE: Login redirects back to login page
SOLUTION:
1. Check credentials (case-sensitive):
   Username: MIT Mysore (capital M, capital M)
   Password: mitm@1234 (lowercase)
2. Enable cookies in browser settings
3. Clear localStorage: F12 → Application → Local Storage → Clear

ISSUE: Timetable generation fails
SOLUTION:
1. Check all subjects have weekly_hours set
2. Ensure faculty assigned to all subjects
3. Verify lab rooms exist for department
4. Check enough time slots (5 days × 6 periods = 30 slots)
5. Review error message in console (F12)

ISSUE: Admin settings not applying
SOLUTION:
1. Verify settings saved (check success message)
2. Restart server to reload settings
3. Check global_settings table in Supabase
4. Ensure department generates NEW timetable (not using old one)
